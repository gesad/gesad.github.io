  1. Git History Plugin

O anseio de qualquer empresa ou equipe para desenvolver um software de qualidade garantida e assegurada continua sendo um grande desafio dentro da Engenharia de Software. Para tanto, a utilização de Ambientes de Desenvolvimento de Software (ADS) e Processos de Desenvolvimento de Softwares (PDS) podem colaborar para uma maior elaboração de todo o processo que implicará num apoio de ferramentas especializadas para o efetivo ciclo de vida de um software. Os ADSs podem ser definidos como ambientes/sistemas que irão objetivar o apoio ao processo de software, apoiando cada atividade desse complexo processo, utilizando diversas ferramentas e facilidades integradas.

É notório que os Sistemas de Controle de Versão são fundamentais no dia-a-dia das equipes de desenvolvimento de software, permitindo que várias pessoas e até diferentes equipes em qualquer lugar do mundo, trabalhem simultaneamente nos mesmos arquivos sem que haja qualquer tipo de perda ou sobrescrita de código-fonte por parte de um desenvolvedor. Possuem também outras funcionalidades, como o versionamento do projeto e a manutenção do histórico de alterações.

Este projeto se deu devido a identificação de problemas enfrentados pelos desenvolvedores de sistemas em acompanhar o andamento das atividades de um projeto. Feito essa análise, buscou-se produzir uma ferramenta que auxiliaria na realização dessas atividades.
